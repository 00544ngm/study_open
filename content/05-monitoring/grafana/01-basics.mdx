## 概述

Grafana 是开源的可视化和监控仪表盘平台，支持 Prometheus、Elasticsearch、CloudWatch 等 30+ 数据源。在 AI Agent 项目中，Grafana 用于展示 **请求量、延迟、错误率、Token 消耗** 等核心指标。

## 安装启动

```bash
# Docker 启动（推荐）
docker run -d \
  --name=grafana \
  -p 3000:3000 \
  -v grafana-data:/var/lib/grafana \
  grafana/grafana:latest

# 访问 http://localhost:3000
# 默认账号/密码: admin / admin（首次登录要求修改密码）
```

```yaml
# 使用 docker-compose 与服务端配套
services:
  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    volumes:
      - grafana-data:/var/lib/grafana
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin123
      - GF_INSTALL_PLUGINS=grafana-piechart-panel
    restart: unless-stopped

volumes:
  grafana-data:
```

## 数据源配置

### 添加 Prometheus 数据源

1. 打开 Configuration → Data Sources → Add data source
2. 选择 Prometheus
3. URL 填写 `http://prometheus:9090`
4. 点击 Save & Test

```yaml
# 通过配置文件预配（推荐生产环境使用）
# provisioning/datasources/prometheus.yml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    url: http://prometheus:9090
    access: proxy
    isDefault: true
    editable: false
```

### 其他常用数据源

```yaml
# provisioning/datasources/loki.yml（日志）
apiVersion: 1
datasources:
  - name: Loki
    type: loki
    url: http://loki:3100
    access: proxy

# provisioning/datasources/elasticsearch.yml
apiVersion: 1
datasources:
  - name: ES-Logs
    type: elasticsearch
    url: http://elasticsearch:9200
    database: "[logstash-]YYYY.MM.DD"
```

## 创建仪表盘

### 通过 UI 创建

1. Dashboards → New Dashboard → Add visualization
2. 选择数据源（如 Prometheus）
3. 在 Query 编辑器中输入 PromQL：

```promql
# Agent 请求量（每秒）
rate(agent_requests_total[5m])

# 按 endpoint 分割
sum by(endpoint) (rate(agent_requests_total[5m]))

# p95 延迟
histogram_quantile(0.95,
  sum by(le) (rate(agent_latency_seconds_bucket[5m]))
)
```

4. 调整图表类型（Time series、Stat、Bar gauge 等）
5. 设置告警阈值

### 常用面板类型

| 面板类型 | 用途 | 适用数据 |
|----------|------|----------|
| Time series | 趋势图 | 请求量、延迟随时间变化 |
| Stat | 单值统计 | 当前错误数、总请求数 |
| Bar gauge | 条形仪表 | 资源利用率 |
| Table | 表格 | 按维度分组统计 |
| Pie chart | 饼图 | 错误类型分布 |

### 通过 JSON 导入（可复用）

```json
{
  "title": "Agent API 监控",
  "panels": [
    {
      "title": "QPS",
      "type": "timeseries",
      "targets": [{
        "expr": "sum(rate(agent_requests_total[5m]))",
        "legendFormat": "QPS"
      }]
    },
    {
      "title": "P95 延迟",
      "type": "timeseries",
      "targets": [{
        "expr": "histogram_quantile(0.95, sum by(le) (rate(agent_latency_seconds_bucket[5m])))",
        "legendFormat": "p95"
      }]
    }
  ]
}
```

也可直接导出为 JSON 文件放在 `provisioning/dashboards/` 目录下自动加载。

## Agent 项目推荐面板

一个完整的 Agent 监控仪表盘应包含：

1. **请求量** — QPS、按 endpoint/agent 分组
2. **延迟** — p50/p95/p99 延迟趋势
3. **错误率** — 错误数、错误率百分比、按类型分组
4. **Token 消耗** — input/output tokens、预估成本
5. **工具调用** — 各工具调用频率、成功率
6. **系统资源** — CPU、内存、连接数

## 告警规则

```yaml
# provisioning/rules/alerts.yml
groups:
  - name: agent-alerts
    rules:
      - alert: HighErrorRate
        expr: |
          sum(rate(agent_errors_total[5m]))
          / sum(rate(agent_requests_total[5m]))
          > 0.05
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "错误率超过 5%"

      - alert: HighLatency
        expr: |
          histogram_quantile(0.95,
            sum by(le) (rate(agent_latency_seconds_bucket[5m]))
          ) > 2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "P95 延迟超过 2 秒"
```
