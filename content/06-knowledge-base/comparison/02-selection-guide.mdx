## 入门

选型没有银弹。本文从对比表、基准测试、成本分析和决策树四个维度，帮你找到最适合业务场景的向量数据库。

## 核心

### 五大向量数据库全景对比

| 特性 | Qdrant | Pinecone | Weaviate | Milvus | Chroma |
|---|---|---|---|---|---|
| **开源** | 开源 + Cloud | 闭源(SaaS) | 开源 + Cloud | 开源 + Cloud | 开源 |
| **部署** | 自托管 / Cloud | 全托管 | 自托管 / Cloud | 自托管 / Cloud | 嵌入式/自托管 |

## 决策树

```
开始选型
│
├─ 数据量 < 100 万？
│   ├─ 要零运维？ ──> Pinecone（全托管，最快上线）
│   ├─ 要免费/本地？
│   │   ├─ 需要混合搜索？ ──> Weaviate（原生混合搜索）
│   │   └─ 追求简洁？ ──> Qdrant（docker up 即用）
│   └─ 仅仅是学习原型？ ──> Chroma（pip install 即用）
│
├─ 数据量 100 万 ~ 1 亿？
│   ├─ 已有 K8s？ ──> Milvus / Qdrant (Helm)
│   ├─ 需要高级过滤能力？
│   │   ├─ 分组搜索 + Payload 过滤 ──> Qdrant
│   │   └─ 混合搜索 + GraphQL 接口 ──> Weaviate
│   └─ 不想管运维？ ──> Pinecone
│
└─ 数据量 > 1 亿？
    ├─ 预算充足？ ──> Milvus（分布式原生，GPU 加速）
    └─ 预算有限？ ──> Qdrant（多节点部署 + 量化压缩）
```

## 成本对比

| 项目 | Qdrant | Weaviate | Milvus | Chroma |
|---|---|---|---|---|
| **服务器** | 1 x 4C 16G | 1 x 8C 32G | 3 x 8C 32G | 1 x 2C 8G |

| **存储** | 100GB SSD | 150GB SSD | 200GB SSD | 50GB SSD |
| **维护工时** | 低 (单机) | 中 | 高 (K8s) | 极低 |

## 自托管 vs 云服务

```python
def deployment_decision(
    data_sensitivity: str,   # "high" | "low"
    team_size: int,
    budget_monthly: float,
    expected_growth: float,  # 月增长率
) -> str:
    """决策自托管还是上云"""

    reasons = []

    if data_sensitivity == "high":
        reasons.append("数据合规需自托管")
        return "自托管"

    if team_size < 3:
        reasons.append("运维人力不足")
        return "云服务"

    cloud_cost = expected_growth * budget_monthly * 0.3
    self_hosted_cost = expected_growth * budget_monthly * 0.1

    if cloud_cost > self_hosted_cost * 3:
        reasons.append(f"云成本（${cloud_cost:.0f}）远超自托管（${self_hosted_cost:.0f}）")
        return "自托管"

    reasons.append(f"云服务性价比可以接受（${cloud_cost:.0f} vs ${self_hosted_cost:.0f}）")
    return "云服务"
```

### 选型总结表

| 考量因素 | 选自托管 | 选云服务 |
|---|---|---|
| 团队规模 | >= 3 人运维 | <= 2 人 / 无运维 |
| 数据敏感度 | 高 (金融/医疗/合规) | 低 (公开数据) |
| 规模预期 | 稳定或可预测 | 快速增长 / 波动大 |
| 预算 | 长期成本优先 | 短期快速上线 |
| 定制需求 | 需要深度调优 | 标准需求即可 |
| 可用性要求 | 可接受有限 SLA | 企业级 SLA |

### 迁移指南

从一个向量库迁移到另一个的核心步骤：

```python
"""向量数据库迁移模板 - 以 Qdrant 迁往 Milvus 为例"""
import json
from qdrant_client import QdrantClient
from pymilvus import connections, Collection, CollectionSchema, FieldSchema, DataType

def migrate_collection(
    source_url: str,
    target_host: str,
    collection_name: str,
    batch_size: int = 100,
):
    # 1. 读取源数据
    src = QdrantClient(source_url)
    scroll_result = src.scroll(
        collection_name=collection_name,
        limit=batch_size,
        with_payload=True,
        with_vectors=True,
    )

    # 2. 创建目标 Schema
    points, _ = scroll_result
    if not points:
        return

    dim = len(points[0].vector)
    fields = [
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
        FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=dim),
        FieldSchema(name="payload", dtype=DataType.JSON),
    ]
    schema = CollectionSchema(fields)
    connections.connect(host=target_host, port="19530")
    collection = Collection(collection_name, schema)

    # 3. 批量迁移
    for i in range(0, len(points), batch_size):
        batch = points[i:i + batch_size]
        entities = [
            [p.id for p in batch],
            [p.vector for p in batch],
            [json.dumps(p.payload) for p in batch],
        ]
        collection.insert(entities)

    # 4. 创建索引
    collection.create_index("vector", {
        "index_type": "IVF_FLAT",
        "metric_type": "COSINE",
        "params": {"nlist": 128},
    })
    collection.load()
    print(f"迁移完成: {len(points)} 条向量")
```

**迁移检查清单：**

- [ ] 向量维度是否一致？（必须相同）
- [ ] 距离算法是否匹配？（Cosine / Dot / Euclid）
- [ ] 元数据 Schema 映射是否完成？
- [ ] 索引参数是否需要重新调优？
- [ ] 数据一致性验证（抽样对比召回率）
- [ ] 双写期间流量切换方案
- [ ] 回滚预案

### 未来趋势

1. **DiskANN 类算法**: 用 SSD + 内存缓存替代纯内存索引，大幅降低成本（Qdrant 已支持）
2. **多模态向量**: 向量数据库原生支持文本、图像、音频多模态 Embedding
3. **SQL 融合**: 对标 PostgreSQL + pgvector，向量查询融入标准 SQL
4. **Serverless**: 按请求付费、零空闲成本的向量服务（Pinecone Serverless、Zilliz Serverless）
