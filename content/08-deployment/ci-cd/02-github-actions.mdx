## 工作流基础

GitHub Actions 是 GitHub 内置的 CI/CD 平台，通过 YAML 工作流定义自动化流水线。

### 工作流文件结构

```yaml
# .github/workflows/ci.yml
name: CI Pipeline                    # 工作流名称

on:                                  # 触发条件
  push:
    branches: [main, develop]
    paths-ignore:                    # 文档变更不触发
      - '**.md'
      - 'docs/**'
  pull_request:
    branches: [main]
  workflow_dispatch:                 # 手动触发
    inputs:
      environment:
        description: '目标环境'
        required: true
        default: 'staging'
        type: choice
        options:
          - staging
          - production

# 环境变量（对所有 job 生效）
env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  # JOB 定义
  build:
    name: Build and Test
    runs-on: ubuntu-latest          # 运行环境
    timeout-minutes: 30             # 超时保护

    # 并发控制
    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
      cancel-in-progress: true

    steps:
    - uses: actions/checkout@v4     # 检出代码
    - run: echo "触发事件: ${{ github.event_name }}"
```

## Matrix 构建

矩阵策略在多种配置组合中并行运行测试，确保跨版本兼容。

```yaml
jobs:
  test:
    name: Test (Python ${{ matrix.python-version }})
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ['3.11', '3.12', '3.13']
        os: [ubuntu-latest, windows-latest]
        # 排除不需要的组合
        exclude:
          - os: windows-latest
            python-version: '3.13'
        # 包含额外组合
        include:
          - os: ubuntu-latest
            python-version: '3.12'
            experimental: false
      # 失败的作业不阻塞其他组合
      fail-fast: false
      # 最大并行数
      max-parallel: 4

    # 继续执行即使某些组合失败
    continue-on-error: ${{ matrix.experimental || false }}

    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with:
        python-version: ${{ matrix.python-version }}
        cache: 'pip'
        cache-dependency-path: '**/requirements*.txt'

    - name: 安装依赖
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
        pip install -r requirements-dev.txt

    - name: 运行测试
      run: pytest tests/ -v --cov=app --cov-report=xml --junitxml=test-results.xml

    - name: 上传测试结果
      if: always()  # 即使失败也上传
      uses: actions/upload-artifact@v4
      with:
        name: test-results-${{ matrix.os }}-${{ matrix.python-version }}
        path: test-results.xml

    - name: 上传覆盖率
      uses: actions/upload-artifact@v4
      with:
        name: coverage-${{ matrix.os }}-${{ matrix.python-version }}
        path: coverage.xml
```

Matrix 配置选型指南：

| 场景 | 策略 |
|------|------|
| 多 Python 版本兼容 | `matrix.python-version: ['3.11', '3.12', '3.13']` |
| 多操作系统 | `matrix.os: [ubuntu-latest, macos-latest, windows-latest]` |
| 多依赖版本 | `matrix.dependency-version: [6, 7]` 配合 `services` |
| 环境变量组合 | `matrix.env: [dev, staging, production]` |

## 缓存策略

合理利用缓存大幅减少流水线执行时间。

### pip 依赖缓存

```yaml
- name: 缓存 pip
  uses: actions/cache@v4
  with:
    path: ~/.cache/pip
    key: ${{ runner.os }}-pip-${{ hashFiles('**/requirements*.txt') }}
    restore-keys: |
      ${{ runner.os }}-pip-
```

### Docker 层缓存

```yaml
- name: 设置 Docker Buildx
  uses: docker/setup-buildx-action@v3

- name: 缓存 Docker 层
  uses: docker/build-push-action@v6
  with:
    context: .
    cache-from: type=gha
    cache-to: type=gha,mode=max
    tags: app:latest
    load: true
```

### npm / pnpm 缓存

```yaml
- name: 缓存 pnpm
  uses: actions/cache@v4
  with:
    path: |
      ~/.pnpm-store
      node_modules
    key: ${{ runner.os }}-pnpm-${{ hashFiles('pnpm-lock.yaml') }}
    restore-keys: |
      ${{ runner.os }}-pnpm-
```

### 缓存命中优化

```yaml
- name: 条件缓存恢复
  uses: actions/cache/restore@v4
  id: cache
  with:
    path: ~/.cache/pip
    key: ${{ runner.os }}-pip-${{ hashFiles('requirements.txt') }}

# 仅在缓存未命中时安装依赖
- name: 安装依赖
  if: steps.cache.outputs.cache-hit != 'true'
  run: pip install -r requirements.txt

# 保存缓存
- name: 保存缓存
  uses: actions/cache/save@v4
  if: steps.cache.outputs.cache-hit != 'true'
  with:
    path: ~/.cache/pip
    key: ${{ runner.os }}-pip-${{ hashFiles('requirements.txt') }}
```

## Docker 构建与推送

完整的镜像构建、签名和推送流水线。

```yaml
jobs:
  build-and-push:
    name: Build & Push Docker Image
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write        # 推送到 GHCR
      id-token: write        # 镜像签名

    steps:
    - uses: actions/checkout@v4

    - name: 设置 Docker Buildx
      uses: docker/setup-buildx-action@v3

    - name: 登录到容器仓库
      uses: docker/login-action@v3
      with:
        registry: ghcr.io
        username: ${{ github.actor }}
        password: ${{ secrets.GITHUB_TOKEN }}

    - name: 生成镜像元数据
      id: meta
      uses: docker/metadata-action@v5
      with:
        images: |
          ghcr.io/${{ github.repository }}
          ${{ secrets.AWS_ACCOUNT }}.dkr.ecr.ap-northeast-1.amazonaws.com/agent-api
        tags: |
          type=sha,prefix=sha-
          type=semver,pattern={{version}}
          type=ref,event=branch
          type=raw,value=latest,enable=${{ github.ref == 'refs/heads/main' }}

    - name: 构建并推送
      uses: docker/build-push-action@v6
      with:
        context: .
        push: true
        tags: ${{ steps.meta.outputs.tags }}
        labels: ${{ steps.meta.outputs.labels }}
        cache-from: type=gha
        cache-to: type=gha,mode=max
        build-args: |
          BUILD_ENV=production
          GIT_SHA=${{ github.sha }}
        # 多平台构建
        platforms: linux/amd64,linux/arm64
        # SBOM 和 Provenance 证明
        provenance: true
        sbom: true
```

## 环境与 Secrets

### 环境保护规则

```yaml
jobs:
  deploy-staging:
    name: Deploy to Staging
    runs-on: ubuntu-latest
    environment:
      name: staging
      url: https://staging.agent.example.com  # 显示在 PR 页面
    # staging 环境自动部署

  deploy-production:
    name: Deploy to Production
    runs-on: ubuntu-latest
    needs: [deploy-staging, integration-test]
    environment:
      name: production
      url: https://api.agent.example.com
    # production 环境需要审核批准
    # 在 GitHub 仓库 Settings > Environments 中配置审核者
```

### 敏感信息管理

```yaml
steps:
  - name: 配置 AWS 凭证
    uses: aws-actions/configure-aws-credentials@v4
    with:
      role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
      aws-region: ap-northeast-1

  - name: 从 Secrets Manager 读取配置
    run: |
      secrets=$(aws secretsmanager get-secret-value \
        --secret-id agent-platform/${{ github.ref_name }} \
        --query SecretString --output text)
      echo "$secrets" | jq -r 'to_entries[] | "\(.key)=\(.value)"' >> .env

  - name: 部署
    run: |
      docker compose --env-file .env up -d
    env:
      # 避免在日志中泄露
      OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
    # 日志脱敏
    # 注意: GitHub Actions 会自动遮蔽结构化 secrets
```

## 部署到云平台

### 部署到 AWS ECS

```yaml
jobs:
  deploy-ecs:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4

    - name: 配置 AWS 凭证
      uses: aws-actions/configure-aws-credentials@v4
      with:
        role-to-assume: ${{ secrets.AWS_DEPLOY_ROLE }}
        aws-region: ap-northeast-1

    - name: 登录 ECR
      id: login-ecr
      uses: aws-actions/amazon-ecr-login@v2

    - name: 构建并推送
      env:
        ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
        ECR_REPOSITORY: agent-api
        IMAGE_TAG: ${{ github.sha }}
      run: |
        docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG .
        docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG

    - name: 部署到 ECS
      uses: aws-actions/amazon-ecs-deploy-task-definition@v2
      with:
        task-definition: task-definition.json
        service: agent-api-service
        cluster: agent-cluster
        wait-for-service-stability: true
        image: ${{ steps.login-ecr.outputs.registry }}/agent-api:${{ github.sha }}
```

### 部署到 GCP Cloud Run

```yaml
jobs:
  deploy-cloud-run:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4

    - id: auth
      uses: google-github-actions/auth@v2
      with:
        workload_identity_provider: ${{ secrets.GCP_WIF_PROVIDER }}
        service_account: ${{ secrets.GCP_SERVICE_ACCOUNT }}

    - name: 设置 Cloud SDK
      uses: google-github-actions/setup-gcloud@v2

    - name: 构建并推送到 Artifact Registry
      run: |
        gcloud builds submit \
          --tag ${{ secrets.GCP_REGION }}-docker.pkg.dev/${{ secrets.GCP_PROJECT }}/agent-api/${{ github.sha }}

    - name: 部署到 Cloud Run
      uses: google-github-actions/deploy-cloudrun@v2
      with:
        service: agent-api
        image: ${{ secrets.GCP_REGION }}-docker.pkg.dev/${{ secrets.GCP_PROJECT }}/agent-api/${{ github.sha }}
        region: ${{ secrets.GCP_REGION }}
        env_vars: |
          LOG_LEVEL=info
          ENVIRONMENT=production
        secrets: |
          OPENAI_API_KEY=openai-api-key:latest
        flags: |
          --min-instances=0
          --max-instances=10
          --cpu=2
          --memory=4Gi
          --concurrency=80
```

## 可复用工作流

将通用流水线抽取为可复用工作流，减少重复。

```yaml
# .github/workflows/reusable-test.yml（被调用方）
name: Reusable Test Suite

on:
  workflow_call:
    inputs:
      python-version:
        required: true
        type: string
      coverage-threshold:
        required: false
        type: number
        default: 80
    secrets:
      CODECOV_TOKEN:
        required: false

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-python@v5
      with:
        python-version: ${{ inputs.python-version }}
    - run: |
        pip install -r requirements.txt
        pip install -r requirements-dev.txt
    - run: pytest --cov=app --cov-report=xml
    - name: 检查覆盖率
      if: inputs.coverage-threshold > 0
      run: |
        coverage=$(python -c "import xml.etree.ElementTree as ET; \
          root=ET.parse('coverage.xml'); \
          print(root.find('.//line_rate').text)")
        echo "覆盖率: $coverage"
        python -c "assert float('$coverage') >= ${{ inputs.coverage-threshold }}, '覆盖率不足'"
```

## 完整的 CI/CD 流水线

将上述所有要素集成为一个完整工作流：

```yaml
# .github/workflows/main.yml
name: Agent Platform CI/CD

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
  workflow_dispatch:
    inputs:
      deploy-to:
        description: '部署环境'
        type: choice
        options: [staging, production]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

env:
  DOCKER_REGISTRY: ghcr.io

jobs:
  lint:
    name: Code Quality
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
          cache: 'pip'
      - run: pip install -r requirements-dev.txt
      - run: ruff check app/ tests/
      - run: ruff format --check app/ tests/
      - run: mypy app/

  test:
    name: Tests (${{ matrix.python-version }})
    needs: lint
    strategy:
      matrix:
        python-version: ['3.11', '3.12']
      fail-fast: false
    runs-on: ubuntu-latest
    services:
      redis:
        image: redis:7-alpine
        ports: ["6379:6379"]
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_PASSWORD: test
          POSTGRES_DB: test
        ports: ["5432:5432"]
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
          cache: 'pip'
      - run: pip install -r requirements.txt -r requirements-dev.txt
      - run: pytest tests/ -v --cov=app --cov-report=xml -n auto
        env:
          REDIS_URL: redis://localhost:6379/0
          DATABASE_URL: postgresql://postgres:test@localhost:5432/test
      - uses: codecov/codecov-action@v5
        with:
          fail_ci_if_error: true

  build:
    name: Build Docker Image
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: docker/setup-buildx-action@v3
      - uses: docker/login-action@v3
        with:
          registry: ${{ env.DOCKER_REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/build-push-action@v6
        with:
          push: true
          tags: |
            ${{ env.DOCKER_REGISTRY }}/${{ github.repository }}:${{ github.sha }}
            ${{ env.DOCKER_REGISTRY }}/${{ github.repository }}:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy-staging:
    name: Deploy to Staging
    needs: build
    if: github.ref == 'refs/heads/develop'
    environment:
      name: staging
      url: https://staging.agent.example.com
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: |
          docker compose -f docker-compose.yml pull
          docker compose -f docker-compose.yml up -d
        env:
          IMAGE_TAG: ${{ github.sha }}

  deploy-production:
    name: Deploy to Production
    needs: [build, deploy-staging]
    if: github.ref == 'refs/heads/main'
    environment:
      name: production
      url: https://api.agent.example.com
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_DEPLOY_ROLE }}
          aws-region: ap-northeast-1
      - run: |
          aws ecs update-service \
            --cluster agent-cluster \
            --service agent-api \
            --force-new-deployment \
            --region ap-northeast-1
```

## 练习

1. 为你的 Agent 项目编写一个完整的 CI 工作流：lint → test (matrix: 3.11, 3.12) → build Docker
2. 添加 Docker 缓存机制，确保第二次运行速度显著提升
3. 配置 CD 工作流：手动触发部署到 staging，自动部署到 production（需环境审批）
4. 将通用测试步骤抽取为可复用工作流，并在两个项目中调用
5. 在 PR 中添加测试覆盖率报告和部署链接（使用环境 URL）
