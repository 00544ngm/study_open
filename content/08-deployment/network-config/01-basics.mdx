## 核心

### 端口与监听

每个网络服务都绑定在一个端口上。部署时最常见的问题是"服务启动了但访问不了"，通常都是端口相关的问题。

```bash
# 查看哪些端口在监听
ss -tlnp
# 或
netstat -tlnp
```

关键理解：

- `0.0.0.0:3000` — 在所有网络接口上监听（外网可访问的前提）
- `127.0.0.1:3000` — 只在本地回环地址监听（仅本机可访问）
- `[::]:3000` — 在 IPv6 所有接口上监听

```bash
# 检查特定端口是否被占用
ss -tlnp | grep 3000
```

### 防火墙管理

#### iptables（Linux 内核防火墙）

```bash
# 查看当前规则
iptables -L -n --line-numbers
iptables -L INPUT -n -v

# 允许 SSH（务必先放行，防止锁掉自己）
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# 允许 HTTP/HTTPS
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 允许应用端口
iptables -A INPUT -p tcp --dport 3000 -j ACCEPT

# 允许已建立的连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 拒绝其他所有入站
iptables -P INPUT DROP
```

**重要**：iptables 规则重启后丢失。持久化需要安装 iptables-persistent：

```bash
apt install -y iptables-persistent
netfilter-persistent save
```

#### ufw（Uncomplicated Firewall）

Ubuntu 推荐的防火墙前端，底层还是 iptables：

```bash
# 启用 ufw（默认拒绝入站）
ufw default deny incoming
ufw default allow outgoing

# 放行常用端口
ufw allow ssh
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 3000/tcp

# 启用防火墙
ufw enable

# 查看状态
ufw status verbose
```

#### firewalld（CentOS/RHEL 默认）

```bash
# 启动并启用
systemctl start firewalld
systemctl enable firewalld

# 放行端口
firewall-cmd --permanent --add-port=3000/tcp
firewall-cmd --reload

# 查看规则
firewall-cmd --list-all
```

### 云服务器安全组

以阿里云为例，安全组是**云层面的防火墙**，优先于 OS 防火墙。即使服务器内部 iptables 全放行，安全组没开端口也一样无法访问。

**配置要点：**

| 协议 | 端口 | 授权对象 | 说明 |
|------|------|---------|------|
| TCP | 22 | 0.0.0.0/0 | SSH 远程登录 |
| TCP | 80 | 0.0.0.0/0 | HTTP |
| TCP | 443 | 0.0.0.0/0 | HTTPS |
| TCP | 3000-3010 | 0.0.0.0/0 | 应用端口范围 |

**安全原则：**

- 仅开放必要的端口
- 管理面（SSH）建议限制源 IP，不使用 0.0.0.0/0
- 数据库端口（3306、5432、6379 等）不要对公网开放
- 定期审查安全组规则，清理未使用的规则

```bash
# 查看实例绑定的安全组（阿里云 CLI）
aliyun ecs DescribeInstanceAttribute --InstanceId iZbp172k6jekk94pkdbcdwZ

# 查看安全组规则
aliyun ecs DescribeSecurityGroupAttribute --SecurityGroupId sg-xxxxx
```

### 端口映射与转发

#### docker 端口映射

```yaml
# docker-compose.yml
services:
  web:
    ports:
      - "3001:3000"   # 宿主机 3001 → 容器内部 3000
```

映射格式：`宿主机端口:容器端口`

- 宿主机端口不能冲突（两个服务不能都用 3000）
- `0.0.0.0:3001:3000` 与 `3001:3000` 等价
- `127.0.0.1:3001:3000` 表示只在本地暴露

#### iptables 端口转发

```bash
# 将 8080 转发到 80
iptables -t nat -A PREROUTING -p tcp --dport 8080 -j REDIRECT --to-port 80
```

#### socat 转发（临时用）

```bash
# 将 3001 转发到 3000
socat TCP-LISTEN:3001,fork TCP:localhost:3000
```

### 网络故障排查流程

当服务"启动但访问不了"时，按这个顺序排查：

```
用户浏览器 → DNS 解析 → 云安全组 → OS 防火墙(iptables/ufw) → 端口监听 → 服务状态
```

**逐层检查命令：**

```bash
# 1. 服务是否运行
docker ps | grep web

# 2. 端口是否监听
ss -tlnp | grep 3000

# 3. 本机能否访问
curl -s http://localhost:3000

# 4. 防火墙是否拦截
iptables -L INPUT -n -v | grep 3000

# 5. 云安全组是否开放（在云控制台查看）

# 6. 外网能否连通
# 在另一台机器或手机上执行：
curl -s http://你的公网IP:3000
# 或
telnet 你的公网IP 3000
```

**常见问题与解决：**

| 现象 | 原因 | 解决 |
|------|------|------|
| `curl localhost:3000` 正常，外网不通 | 安全组没开端口 | 去云控制台添加入方向规则 |
| `curl localhost:3000` 正常，`curl 公网IP:3000` 卡住 | 服务只监听 127.0.0.1 | 改为监听 0.0.0.0 |
| `ss -tlnp` 看不到端口 | 服务没启动或崩溃 | `docker logs` 查看原因 |
| 端口冲突 | 两个服务用了同个端口 | 改端口或停掉冲突的服务 |
| iptables 重启后规则丢失 | 没持久化 | 安装 iptables-persistent |

## 练习

1. 在服务器上执行 `ss -tlnp`，列出当前所有监听端口并说明每个端口的用途
2. 用 ufw 配置防火墙：放行 22、80、443 端口，拒绝其他入站，然后验证规则生效
3. 启动一个 Nginx 容器映射到宿主机 8080 端口，确认从外网可以访问
4. 模拟排查：故意把 docker 端口映射改成 `127.0.0.1:3000:3000`，观察外网是否还能访问，然后修复
