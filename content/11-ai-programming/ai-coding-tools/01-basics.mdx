## 入门

### 什么是 AI 编程工具

AI 编程工具 = 大语言模型 + 代码理解 + 文件操作能力。它们不只是"自动补全"，而是能理解项目结构、跨文件修改、执行命令的智能体。

### 主流工具对比

| 工具 | 模式 | 适合场景 | 局限性 |
|------|------|---------|--------|
| **Claude Code** | Agent 模式 | 复杂重构、跨文件修改、CI 集成 | 需要终端环境 |
| **Cursor** | IDE + Tab 补全 | 日常编码、快速问答 | 依赖 VS Code 生态 |
| **Copilot** | IDE 内嵌补全 | 行级补全、简单生成 | Agent 能力弱 |
| **Aider** | Git 原生 Agent | 版本管理一体化 | 无 IDE 界面 |

### Claude Code 核心工作流

```bash
# 在项目目录启动
claude

# 用自然语言描述需求
"给这个 API 端点添加输入验证"

# Claude Code 会自动：
# 1. 读取相关文件理解上下文
# 2. 规划修改方案
# 3. 执行修改并显示 diff
# 4. 等待你确认后应用
```

### 三种交互模式

**模式 1：对话模式（默认）**
```
$ claude
> 分析 src/api/handler.ts 的性能瓶颈
→ AI 读取文件 → 分析 → 输出结果
```

**模式 2：单次命令模式**
```bash
# 执行一次后退出，适合 CI
claude --print "解释这段代码" < main.py
```

**模式 3：管道模式**
```bash
# 从 stdin 传入 prompt
echo "优化这个函数的性能" | claude --model claude-opus-4-7
cat error.log | claude "分析这些错误"
```

### 核心概念

- **Agent 模式**：自动规划→执行→验证循环，AI 自主决定下一步操作
- **文件编辑**：精确的行级插入/替换，而非重写整个文件
- **命令执行**：在安全沙箱中运行 shell 命令并捕获输出
- **上下文窗口**：AI 能看到当前对话中的所有内容，但有限制（通常 100K-200K tokens）

> **知识关联**：AI 编程工具的核心能力——文件操作和命令执行——在[编程智能体构建](/12-building-coding-agents/file-operations/01-basics)中有深入讲解；提示词工程详见[编码提示词](/11-ai-programming/code-prompt/01-basics)。

### 实际使用场景

```
场景 1：调试
  "这个测试用例挂了，帮我找出原因并修复"
  → AI 运行测试 → 分析报错 → 定位代码 → 修复

场景 2：重构
  "把这个函数拆分成三个小函数"
  → AI 理解代码 → 设计拆分方案 → 修改 → 验证

场景 3：写测试
  "给 src/utils/format.ts 写单元测试"
  → AI 分析函数 → 生成测试用例 → 运行验证
```
