## 项目：AI 辅助开发的完整流程

### 目标

用 Claude Code / Cursor 构建一个实用的命令行工具：**批量文件重命名工具**，支持正则匹配、预览、回滚。

### 流程

#### 阶段 1：需求分析与架构设计

给 AI 的初始 prompt：

```
帮我设计一个批量文件重命名 CLI 工具，要求：
- 支持正则匹配和替换
- 执行前预览修改结果
- 支持回滚操作
- Python 实现，使用 argparse
- 类型注解完整

先输出架构设计，确认后再写代码。
```

**审查点**: AI 的架构是否合理？模块划分是否清晰？

#### 阶段 2：核心实现

分步 prompt：

```
1. 实现文件扫描模块：递归扫描目录，返回匹配正则的文件列表
2. 实现重命名引擎：执行实际的重命名操作
3. 实现回滚模块：记录修改日志，支持 undo
4. 实现 CLI 接口：argparse 解析参数，连接各模块
```

**审查点**: 每个模块提交后，审查 diff 再继续下一步。

#### 阶段 3：测试覆盖

```
为上面实现的重命名工具编写 pytest 测试：
- 测试文件扫描（空目录、嵌套目录、匹配模式）
- 测试重命名（正常、冲突、特殊字符）
- 测试回滚（单次回滚、多次回滚）
- 测试 CLI 参数解析
```

**审查点**: 运行 `pytest`，确保所有测试通过。

#### 阶段 4：AI 代码审查

```
审查我之前写的文件重命名工具的代码，检查：
1. 安全性：有没有路径穿越漏洞？
2. 健壮性：并发执行会出问题吗？
3. 可用性：错误信息清晰吗？
4. 性能：处理 10 万个文件会怎样？
```

#### 阶段 5：Git 管理

```bash
# 按流程分步提交
git add src/renamer/scanner.py
git commit -m "feat: 实现文件扫描模块"

git add src/renamer/engine.py
git commit -m "feat: 实现重命名引擎"

git add tests/
git commit -m "test: 添加完整测试覆盖"

# 审查 AI 的提交历史
git log --oneline
```

### 验收标准

- [ ] 工具可以正常运行 `python renamer.py --dry-run ".*\.txt" "new_" ./test`
- [ ] `--dry-run` 显示预览但不实际修改
- [ ] 回滚功能支持恢复到任意历史状态
- [ ] 测试覆盖 > 80%
- [ ] 所有类型注解完整
