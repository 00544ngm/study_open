## 项目：用 AI 重构遗留代码

### 目标

找一个你之前写的"面条式"代码，用 AI 工具进行系统性的架构重构。实践架构原则 + AI 协作的完整工作流。

### 流程

#### 阶段 1：代码分析

```
分析 src/legacy/main.py 的代码结构：
1. 列出所有函数和它们的依赖关系
2. 识别代码坏味道（过长函数、重复代码、职责混乱）
3. 给出重构建议和优先级
```

**输出**: 代码分析报告 + 重构计划

#### 阶段 2：测试先行（关键！）

重构前先写测试，确保重构不破坏功能：

```
分析 src/legacy/main.py 的功能，然后编写测试用例：
- 使用 pytest
- 覆盖所有主要功能路径
- 对于每个测试，描述它在测什么
```

**输出**: `tests/test_legacy_main.py`（所有测试初始应为绿色）

#### 阶段 3：渐进式重构

```
按以下步骤重构 src/legacy/main.py：

Step 1: 将配置提取为 Config 类
Step 2: 将数据库操作提取为 Repository 类
Step 3: 将业务逻辑提取为 Service 类
Step 4: 主流程只负责编排

每步完成后运行测试，确保不红。
```

**审查要点**:
- 每步重构后运行 `pytest` 确认测试通过
- 审查 AI 的命名和抽象是否合理
- 检查有没有过度工程

#### 阶段 4：架构评审

```
分析重构后的代码架构：
1. 类图和依赖关系
2. 是否符合 SOLID 原则
3. 是否有改进空间
4. 与重构前的对比
```

**输出**: 架构对比报告

### 验收标准

- [ ] 重构后代码通过所有原有测试
- [ ] 代码模块化（Config / Repository / Service 分离）
- [ ] 类型注解完整
- [ ] AI 生成的代码有人工审查记录
- [ ] 提交历史清晰展示重构步骤
