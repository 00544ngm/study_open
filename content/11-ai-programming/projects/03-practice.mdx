### 练习 2：分步迭代开发

```python
# 使用分步 Prompt 构建一个 Markdown 转 HTML 转换器

# Round 1：需求分析
round_1_prompt = """
我需要一个 Markdown 转 HTML 的转换器。
先帮我分析需求：
1. 需要支持哪些 Markdown 语法？
2. 建议的模块划分？
3. 技术选型建议？

等我确认后再开始实现。
"""

# Round 2：核心实现
round_2_prompt = """
按照确认的方案，实现以下模块：

Step 1: 标题转换（# → <h1>）
Step 2: 列表转换（- → <ul><li>）
Step 3: 代码块转换（``` → <pre><code>）
Step 4: 链接转换（[text](url) → <a>）

每步完成后等待我审查。
"""

# Round 3：错误处理
round_3_prompt = """
为转换器添加错误处理：
- 未闭合的代码块
- 嵌套过深的列表（超过 6 层）
- 非法的 URL 格式
- 空输入
"""

# Round 4：测试覆盖
round_4_prompt = """
为转换器编写 pytest 测试：
- 每种语法类型的正常情况
- 混合内容（标题 + 列表 + 代码块）
- 边界情况（空行、特殊字符）
- 错误输入
"""

# 你的任务：
# 1. 依次执行 Round 1-4
# 2. 记录 AI 在每一步的输出
# 3. 在 Round 之间加入你的审查和纠正
