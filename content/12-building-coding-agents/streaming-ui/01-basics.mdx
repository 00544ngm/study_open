## 核心

### 为什么终端 UI 很重要

AI 编程智能体的输出是流式的——文字一个字一个字出现，同时还要处理用户的中断和确认、命令执行日志、文件修改 diff。这不是普通的聊天 UI，而是**多路复用终端**。

### SSE 流式传输

服务端通过 Server-Sent Events 将 AI 输出逐块推送到客户端：

```python
from fastapi import StreamingResponse

async def agent_stream():
    for chunk in llm_response():
        yield f"data: {chunk}\n\n"
    yield "data: [DONE]\n\n"

@app.get("/chat/stream")
async def stream():
    return StreamingResponse(agent_stream(),
                             media_type="text/event-stream")
```

### 终端中的流式展示

```typescript
// 逐字符渲染，支持用户中断
async function streamToTerminal(reader: ReadableStreamDefaultReader) {
  const decoder = new TextDecoder();
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    process.stdout.write(decoder.decode(value));
  }
}
```

### ANSI 转义码

终端通过 ANSI 转义序列控制颜色、光标位置、清屏等：

```python
# 基本颜色
RED = "\033[31m"
GREEN = "\033[32m"
YELLOW = "\033[33m"
BLUE = "\033[34m"
RESET = "\033[0m"

# 光标控制
UP = "\033[A"       # 光标上移一行
HIDE = "\033[?25l"  # 隐藏光标
SHOW = "\033[?25h"  # 显示光标
CLEAR = "\033[2J"   # 清屏

# 在终端显示绿色成功消息
print(f"{GREEN}✓ 操作成功{RESET}")

# 原地更新进度条
print(f"\r进度: 50%", end="", flush=True)
```

### 多路复用

同时显示 LLM 输出 + 命令执行日志 + 文件修改 diff，互不干扰：

```
┌── LLM ──────────────────────────────┐
│ 正在分析...                          │
│ 发现 3 个需要修改的位置               │
├── 日志 ──────────────────────────────┤
│ $ npm test                           │
│ PASS src/agent.test.ts               │
├── Diff ──────────────────────────────┤
│ + function validate() {              │
│ - function old_validate() {          │
└───────────────────────────────────────┘
```

这需要设计合理的 UI 分区和输出优先级策略。
