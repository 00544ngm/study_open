Kubernetes 基础

入门

Kubernetes (K8s) 是容器编排平台，用于自动化部署、扩展和管理容器化应用。

核心

核心概念

# Node: 工作节点
# Pod: 最小部署单元（一个或多个容器）
# Deployment: 管理 Pod 副本和滚动更新
# Service: 提供稳定的网络访问入口
# ConfigMap / Secret: 配置管理

部署 Agent API

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: agent-api
  labels:
    app: agent-api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: agent-api
  template:
    metadata:
      labels:
        app: agent-api
    spec:
      containers:
      - name: agent
        image: agent-api:latest
        ports:
        - containerPort: 8000
        env:
        - name: REDIS_URL
          valueFrom:
            secretKeyRef:
              name: app-secrets
              key: redis-url
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 10

Service 暴露

# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: agent-api-service
spec:
  selector:
    app: agent-api
  ports:
  - port: 80
    targetPort: 8000
  type: LoadBalancer

# 部署
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
kubectl get pods -w
kubectl logs -l app=agent-api --tail=100

进阶

自动扩缩容

# hpa.yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: agent-api-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: agent-api
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70

ConfigMap 和 Secret

# config.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: agent-config
data:
  LOG_LEVEL: "info"
  MAX_TOKENS: "4096"
---
apiVersion: v1
kind: Secret
metadata:
  name: app-secrets
type: Opaque
stringData:
  openai-key: "sk-xxx"
  redis-url: "redis://redis-svc:6379"

常用命令

# 排查问题
kubectl describe pod agent-api-xxx
kubectl logs agent-api-xxx -c agent
kubectl exec -it agent-api-xxx -- sh
kubectl port-forward svc/agent-api-service 8000:80

# 查看资源
kubectl top pods
kubectl get events --sort-by='.lastTimestamp'
